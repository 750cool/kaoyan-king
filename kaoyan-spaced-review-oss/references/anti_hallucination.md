# 反幻觉与引用规范

本规范用于从教材 PDF、讲义、真题解析中抽取知识点、公式、定理、例题和复习卡片。核心原则：能回到原文的内容才标为教材来源；模型补充的推导、解释和变式必须明确标注；无法核实的位置、页码、例题号、定理名一律写 `TBD` 或 `未核实`，不得猜。

## 1. 来源标注的最低要求

每条卡片、公式、例题或知识点至少记录以下来源字段：

- `source_title`：教材或资料名。若用户未提供正式书名，写文件名或资料夹名。
- `edition_or_version`：版次、年份、阶段资料名；未知写 `未知版本`。
- `pdf_path`：相对 skill 根目录的路径，例如 `assets/textbooks/higher_math/...pdf`。
- `chapter`：章名或主题。仅在 PDF 中确认后填写；未确认写 `TBD`。
- `section`：节名、小节名或知识块。未确认写 `TBD`。
- `page`：PDF 阅读器页码或书籍印刷页码，必须注明口径。
- `locator`：例题号、定理号、公式号、页内标题、题号等可定位信息；无编号写 `无编号，页内关键词：...`。
- `verified_at`：核验日期，格式 `YYYY-MM-DD`。
- `verified_by`：执行核验的 agent 或角色名，例如 `Source Librarian`、`Hallucination Auditor`。
- `extraction_confidence`：`high`、`medium`、`low`。扫描版、OCR 公式、跨页例题默认不高于 `medium`。
- `ocr_status`：`native_text`、`ocr_checked`、`ocr_low_confidence`、`manual_check_required`。

页码必须区分两类：

- `pdf_page`：PDF 文件的第几页，从阅读器或工具返回结果确认。
- `print_page`：书籍页面上印刷的页码，只有肉眼或 OCR 明确看到时才填。

推荐写法：

```markdown
来源：张宇高数讲义（2027，用户 PDF），`assets/textbooks/higher_math/zy-2027.pdf`，第 3 章第 2 节，PDF p.57 / 印刷 p.49，例 3.12，2026-05-23 由 Hallucination Auditor 核验。
```

若只能确认章节但不能确认页码：

```markdown
来源：张宇高数讲义（版本待核实），第 3 章第 2 节，页码 TBD，例题号 TBD；只确认主题归属，未确认具体页码。
```

PDF 解析与 OCR 质检 Agent 标记为低置信度时，不得直接生成最终卡片；只能生成待核验草稿，直到 Hallucination Auditor 或人工回查确认。

## 2. 三类内容必须分层标注

抽取时把内容分为三层，不能混写成同一种“教材说法”。

### A. 教材原意

指 PDF 中直接出现或忠实改写的定义、公式、定理条件、例题目标、解法步骤和结论。

要求：

- 必须附来源定位。
- 只能做短 paraphrase，不整段照抄。
- 条件、符号、适用范围不得省略成更强结论。
- 不得把教材没有命名的结论擅自命名为“某某定理”。

标注：

```markdown
【教材原意】...
来源：...
```

### B. 模型推导

指为了帮助记忆或讲解，由模型补出的中间步骤、直观解释、等价变形、证明草图、错误辨析。

要求：

- 明确写为模型推导，不冒充教材步骤。
- 与教材结论不一致时，必须暂停并回查 PDF 或请求人工确认。
- 公式变形要逐行检查量纲、条件、符号定义和边界情况。

标注：

```markdown
【模型推导】基于教材公式 ... 推得 ...
依据：教材来源 ...
校验：已检查符号含义和适用条件 / 待二次校验
```

### C. 自编变式

指根据教材例题、考研题型或知识点自行生成的新题、替换参数、改变问法、综合多个知识点的练习。

要求：

- 不得称为“书上例题”“原题”“真题”，除非 PDF 中确有对应原题。
- 需说明变式来自哪个教材例题或知识点类型。
- 改参数时确认不会改变题目可解性和答案结构。
- 难度、考频、陷阱可以由模型评估，但必须标注为模型判断。

标注：

```markdown
【自编变式】...
改编自：教材例 3.12 的题型/方法，不是原题。
模型判断：难度 3/5，考频 4/5。
```

## 3. 二次校验流程

任何进入 `data/decks/` 的结构化卡片，至少完成一次自查；涉及页码、例题号、定理名、公式编号的内容，必须完成二次校验。

### 第一次校验：抽取者自查

抽取者生成卡片后逐项检查：

- 公式左右两边符号是否与教材一致。
- 条件是否完整，例如连续性、可导性、收敛区间、矩阵维数、初始条件、系统因果/稳定条件。
- 变量名是否被误换，例如 `w`、`\omega`、`s`、`z`、`n`、`t`。
- 例题是否真的来自 PDF，而不是由模型补全。
- `source` 字段是否足以让另一个人找到原文。
- 若 OCR 有乱码或公式识别不完整，标记 `needs_pdf_recheck: true`。

### 第二次校验：独立复核

Hallucination Auditor 或另一个 agent 复核时：

- 重新打开 PDF，而不是只看抽取者的摘录。
- 用章节标题、页内关键词、例题号、公式号交叉定位。
- 随机抽查同一批次至少 20% 的普通卡片；公式、定理、例题卡片 100% 复核来源定位。
- 对照原文检查是否把充分条件写成必要条件，或把特殊情形推广成一般结论。
- 对例题卡片检查题干类型、已知条件、求解目标和结论是否匹配。
- 在卡片或批次日志中写明 `verified_at`、`verified_by`、`verification_status`。

可用状态：

- `verified`：来源、页码/定位、公式和条件均已确认。
- `partial`：只确认教材主题或章节，页码/例题号尚未确认。
- `needs_pdf_recheck`：OCR、页码、公式或题号存在疑点，必须回查。
- `model_generated`：模型自编解释或变式，不应当被当作教材原文引用。

## 4. 必须回查 PDF 的情况

出现以下任一情况，不得凭记忆或常识继续填写，必须回查 PDF：

- 需要填写页码、例题号、定理号、公式号、题号。
- 用户要求“按教材”“书上例题”“原文位置”“第几页”。
- 内容涉及容易混淆的条件：一致连续、绝对收敛、条件收敛、可交换极限、矩阵可逆、特征值重数、傅里叶/拉普拉斯/Z 变换收敛域。
- OCR 输出含乱码、缺公式、上下标错位、分式结构不清。
- 同一公式在不同教材版本或章节中符号约定可能不同。
- 模型想补全一个看似熟悉的定理名、例题号、页码或章节名。
- 发现教材原意与模型常识、网络记忆、其他讲义说法不一致。
- 要把知识点加入长期复习卡组或作为后续计划的依据。
- 用户上传了新版本 PDF，旧版页码或例题号可能失效。
- 从截图、目录、笔记片段推断整页内容时。

回查后仍不能确认时，写清楚不确定性：

```markdown
页码：TBD（已查 PDF 目录和第 3 章附近，未定位到该例题号）
状态：needs_pdf_recheck
处理：暂不写入 verified 卡组；可作为 model_generated 临时练习使用
```

## 5. 禁止行为

以下行为会污染复习系统，禁止使用：

- 编造或猜测 PDF 页码、印刷页码、例题号、题号、公式号。
- 把“常见教材一般这么讲”写成“本教材写在第 X 页”。
- 把模型自编题写成“书上例题”“教材原题”“真题”。
- 把用户未提供的教材版本、作者、年份当作已确认信息。
- 给教材没有命名的结论擅自加权威名称。
- 省略关键条件后仍引用教材作为来源。
- 用相邻章节或其他资料的页码替代当前 PDF 页码。
- 只凭文件名或目录推断例题内容。
- 在 OCR 乱码中猜公式结构并标为 verified。
- 为了让卡片看起来完整而补齐未知字段。

## 6. 公式和例题抽取规范

### 公式

公式卡片必须包含：

- 原公式或忠实 paraphrase。
- 符号说明。
- 适用条件。
- 来源定位。
- 常见误用或边界情况。
- 若有推导步骤，区分教材步骤与模型补充步骤。

如果只能确认公式属于通用知识，但未在 PDF 定位：

```markdown
source: "通用知识，教材页码未核实"
verification_status: "partial"
notes: "不得写作教材 p.X；写入正式卡组前需回查 PDF。"
```

### 例题

例题卡片必须包含：

- 题干目标的概括，不长篇复制原文。
- 已知条件和求解对象。
- 教材解法主线。
- 例题号或页内定位；没有编号则写页内关键词。
- 变式题时明确“不是原题”。

例题变式模板：

```markdown
【教材原意】例题方法：...
来源：...

【自编变式】...
改编说明：保留原例题的核心方法，将 ... 改为 ...；不是教材原题。
答案依据：模型推导，待二次校验 / 已二次校验。
```

## 7. 输出前检查清单

发布任何抽取结果前，逐项确认：

- 是否每条教材来源都有可定位字段。
- 是否把 `pdf_page` 与 `print_page` 分开。
- 是否所有未知页码、例题号、定理名都写为 `TBD` 或 `未核实`。
- 是否明确区分【教材原意】【模型推导】【自编变式】。
- 是否避免大段复制教材原文。
- 是否对公式条件、符号、适用范围做了检查。
- 是否标出 `verification_status`。
- 是否列出需要回查 PDF 的遗留项。

## 8. 推荐卡片字段补充

在 `references/card_schema.md` 的基础上，建议为需要来源审计的卡片增加：

```json
{
  "source_title": "TBD",
  "edition_or_version": "TBD",
  "pdf_path": "assets/textbooks/...",
  "pdf_page": "TBD",
  "print_page": "TBD",
  "locator": "TBD",
  "content_origin": "textbook_intent | model_derivation | self_made_variant",
  "verification_status": "partial | verified | needs_pdf_recheck | model_generated",
  "verified_at": "YYYY-MM-DD",
  "verified_by": "Hallucination Auditor",
  "source_notes": "说明页码口径、未核实项、变式来源等"
}
```

`content_origin` 的含义：

- `textbook_intent`：教材原意，必须有来源定位。
- `model_derivation`：模型推导，必须说明依据的教材结论。
- `self_made_variant`：自编变式，必须说明改编自何种题型或知识点。

## 9. 处理不确定性的固定话术

当来源不完整时，优先诚实说明：

- “我目前只确认到章节主题，尚未确认具体页码和例题号。”
- “这一步是模型补充推导，不是教材原文步骤。”
- “该题是根据教材例题方法自编的变式，不是书上原题。”
- “OCR 未能可靠识别该公式，需要回查 PDF 图片页。”
- “不能确认该名称是否为教材中的定理名，暂写为知识点描述。”

宁可留下 `TBD`，也不要填一个看似完整但未经核验的引用。
