# 多 Agent 工作流

本文定义 `kaoyan-spaced-review` skill 在大规模教材解析、抽卡、出题、复习调度和讲解批改时的协作方式。默认使用中文输出；公式、字段名、文件路径和评分标签可保留英文。

## 总原则

- 先定范围，再分工；先验证来源，再生成卡片；先批改用户答案，再更新复习间隔。
- 能按学科、章节、资料来源拆开的任务优先并行；依赖用户答题结果、调度状态或校验结论的任务必须串行。
- 每个 agent 只写自己负责的产物；共享文件发生冲突时，先产出临时草稿或补丁说明，由总规划 agent 合并。
- 所有教材事实必须有来源锚点：书名或资料名、章节、页码或例题号；无法确认时标记 `source_status: unverified`，不得伪造页码。
- 抽卡、变式题和讲解都必须区分“教材原始内容”“教材例题改写”“模型生成变式”。

## Agent 角色

### 总规划 Agent

职责：把用户目标拆成阶段计划、周计划、日任务和 agent 分工。

输入：

- 用户目标、考试年份、可用天数、每日学习时长
- `references/study_planning.md`
- `references/source_manifest.md`
- 现有 `data/plans/`、`data/logs/`、`data/decks/`

输出：

- 阶段计划、周计划、今日任务清单
- 本轮需要启动的 agent 列表、每个 agent 的输入范围和截止产物
- 合并决策记录：采用哪些卡片、推迟哪些章节、哪些来源仍缺失

注意：

- 控制每日负荷：到期复习优先于新卡；欠账过多时减少新卡。
- 不直接批量改卡片内容，除非任务明确要求小规模修订。

### PDF 解析与 OCR 质检 Agent

职责：在教材解析前检查 PDF 是否适合直接抽取，建立页码映射，判断是否需要 OCR、截图识别或人工确认。

输入：

- `assets/textbooks/` 或 `assets/materials/` 中的 PDF
- `references/pdf_ingestion.md`
- `references/source_manifest.md`

输出：

- PDF 清单：文件名、路径、页数、大小、是否有文本层、抽取质量等级
- 页码映射：`pdf_page` 与 `print_page` 的对应关系，无法确认时写 `TBD`
- 低置信度区域：公式密集页、扫描模糊页、例题编号疑似错读页、跨页题目
- 推荐抽取策略：直接文本抽取、OCR、截图核验、人工确认、跳过

串行依赖：

- 必须先于教材解析 Agent 大规模抽取。
- 幻觉校验 Agent 应优先复查它标记为低置信度的页面。

注意：

- 不把 OCR 结果直接当作教材事实；公式、页码、例题号必须二次校验。
- 对数学公式中的上下标、积分限、求和限、矩阵行列、变换符号给出额外低置信度标记。

### 教材解析 Agent

职责：从 PDF 或材料中提取结构化教材事实，为抽卡和出题提供可靠来源。

输入：

- `assets/textbooks/` 或 `assets/materials/` 中指定资料
- `references/source_manifest.md`
- `references/anti_hallucination.md`

输出：

- 章节目录、知识点列表、定理/公式/例题索引
- 可供抽卡使用的来源片段摘要，避免长段原文
- 来源问题清单：缺页、扫描不清、页码错位、版本不明

串行依赖：

- 必须先于数学抽卡、信号抽卡、出题变式和幻觉校验的大部分工作。

### 数学抽卡 Agent

职责：为数学一内容生成主动回忆卡，覆盖高等数学、线性代数、概率论与数理统计。

输入：

- 教材解析 Agent 输出
- `references/card_schema.md`
- 当前阶段计划和已有数学 decks

输出：

- 符合卡片 schema 的数学卡片草稿
- 每张卡的知识点、条件、符号含义、常见陷阱、推导提示、例题锚点
- 与旧卡重复或可合并的建议

要求：

- 公式卡必须写清适用条件；定理卡必须写清结论成立的前提。
- 例题改写不得保留大段原题文字，只保留结构、方法和必要数据。

### 信号抽卡 Agent

职责：为信号与系统内容生成主动回忆卡，覆盖连续/离散信号、系统性质、卷积、傅里叶、拉普拉斯、Z 变换和采样等主题。

输入：

- 教材解析 Agent 输出
- `references/card_schema.md`
- 当前阶段计划和已有信号 decks

输出：

- 符合卡片 schema 的信号卡片草稿
- 每张卡的定义、变换对、性质条件、计算步骤、易混点、例题锚点
- 与数学内容交叉的提醒，例如微分方程、复变函数、级数展开

要求：

- 变换性质必须写明收敛域、因果性、周期性或离散/连续条件。
- 图像类题目如暂不能渲染，应给出文字化的信号描述和待补图标记。

### 出题变式 Agent

职责：基于已验证卡片和教材例题生成练习题、填空题、推导题、综合题和例题变式。

输入：

- 已通过或待校验的卡片
- 教材解析 Agent 的例题索引
- 用户近期错题和薄弱标签

输出：

- 题目、答案、评分点、提示梯度、难度等级
- 变式来源说明：原例题结构、改变的条件、考察点
- 对应卡片 ID 或知识点标签

串行依赖：

- 必须在抽卡后进行；若用于当日练习，应先经过幻觉校验或至少标明 `variant_status: draft`。

### 复习调度 Agent

职责：根据到期卡、遗忘曲线、阶段计划和用户反馈生成每日复习任务，并更新下一次复习日期。

输入：

- `data/decks/*.json`
- `data/logs/`
- 今日可用时间和总规划 Agent 的阶段目标
- 用户答题后的评分：`again`、`hard`、`good`、`easy`

输出：

- 今日复习队列：到期卡、弱项回炉、新卡、长题
- 更新后的间隔、到期日期和日志记录
- 明日预测：预计到期数、新卡建议、风险提醒

串行依赖：

- 每日开始时先生成队列；用户答题和批改完成后再更新 deck。
- 不在批改前预先把卡片标为掌握。

### 批改讲解 Agent

职责：批改用户答案，给出分数、复习评级、关键讲解和追问。

输入：

- 用户作答
- 对应卡片、标准答案、评分点
- 教材锚点和出题变式说明

输出：

- 评级：`again`、`hard`、`good`、`easy`
- 错因分类：概念、条件、计算、符号、步骤、审题、记忆
- 简明讲解、必要推导、下一问或同类巩固题

要求：

- 先指出用户答案中可得分的部分，再定位错误。
- 不把讲解写成大段教材摘录；优先用步骤化推理和短例子。

### 幻觉校验 Agent

职责：独立核查教材事实、公式、页码、例题号、卡片字段和变式题答案，降低幻觉风险。

输入：

- 教材解析结果
- 抽卡草稿、变式题草稿、讲解草稿
- `references/anti_hallucination.md`

输出：

- 校验结论：`pass`、`needs_fix`、`unverified`
- 问题列表：来源不匹配、公式条件缺失、页码可疑、答案不唯一、版权风险
- 修复建议或需要人工确认的事项

冲突权限：

- 对来源、公式、答案正确性拥有阻断权。若校验为 `needs_fix`，相关卡片或题目不得进入正式 deck；只能进入草稿区或待确认清单。

### 资料归档 Agent

职责：维护资料清单、文件位置、版本说明、生成物索引和每日日志，使后续 agent 能快速接续。

输入：

- 新增 PDF、讲义、真题、用户上传材料
- 各 agent 的输出路径和状态
- 当日复习结果和总规划 Agent 的合并决策

输出：

- 更新后的 `references/source_manifest.md` 建议或补丁说明
- `data/logs/` 日志条目
- 资料状态表：已解析、待 OCR、待校验、已入卡、废弃

要求：

- 只归档事实状态，不替其他 agent 重写内容。
- 对同名不同版本资料标注版本、来源和日期，避免后续引用混乱。

## 并行与串行关系

### 可并行执行

- 教材解析可按学科、书籍、章节并行。
- 数学抽卡与信号抽卡可并行，但各自只能处理已解析或明确可定位的来源。
- 出题变式可按卡片组或薄弱标签并行。
- 幻觉校验可按文件、章节或卡片批次并行。
- 资料归档可与抽卡、校验并行收集状态，但最终归档应在合并后完成。

### 必须串行执行

1. 总规划确定范围和优先级。
2. 教材解析建立来源锚点。
3. 数学抽卡/信号抽卡生成草稿。
4. 幻觉校验核查事实、条件和来源。
5. 出题变式基于通过校验的卡片生成练习。
6. 复习调度生成今日队列。
7. 用户作答。
8. 批改讲解给出评级。
9. 复习调度根据评级更新间隔和日志。
10. 资料归档记录产物、状态和缺口。

## 输入输出约定

### 任务输入

每个 agent 启动前应收到最小必要上下文：

- 任务目标：例如“解析同济高数第 3 章极限与连续”
- 范围边界：资料路径、章节、页码、卡片数量上限
- 输出格式：卡片 JSON、Markdown 表格、问题清单或讲解文本
- 质量要求：是否必须有页码、是否可生成 draft、是否要等待校验

### 产物状态

- `draft`：可供后续 agent 处理，但不得直接推给用户背诵。
- `checked`：通过幻觉校验，可进入正式复习或题目队列。
- `needs_fix`：存在明确错误或缺字段，退回原 agent 修改。
- `unverified`：暂时无法核实，可保留但必须醒目标注。
- `archived`：已归档，后续引用应通过 manifest 或日志查找。

### 推荐交接格式

```markdown
## Agent 交接
- 角色：
- 输入范围：
- 已完成：
- 产物路径或内容：
- 来源锚点：
- 风险/待确认：
- 建议下一步：
```

## 冲突处理

- 来源冲突：同一公式、定义或例题在不同资料中不一致时，优先采用用户指定主教材；记录补充资料差异，不擅自合并。
- 卡片重复：若新卡与旧卡知识点相同但提示角度不同，保留更适合主动回忆的一张，另一张合并为变式或补充字段。
- 调度冲突：当新卡计划与大量 overdue 卡冲突时，复习调度 Agent 优先清理 overdue；总规划 Agent 调整阶段节奏。
- 校验冲突：幻觉校验 Agent 与抽卡/出题 Agent 意见不一致时，以可复查来源为准；无来源证明的一方降级为 `unverified`。
- 文件冲突：多人并行修改同一 deck 或 manifest 时，不覆盖对方改动；先输出独立 patch、追加日志或待合并清单，由总规划 Agent 或人工合并。
- 用户偏好冲突：用户临时要求加量、换科目或跳过讲解时，保留复习调度建议，但以用户最新指令执行，并记录风险。

## 每日运行流程

1. 资料归档 Agent 检查是否有新资料、新日志或未合并产物。
2. 总规划 Agent 根据阶段计划、昨日完成情况和今日可用时间确定目标。
3. 复习调度 Agent 从 decks 中选择到期卡、弱项回炉卡和少量新卡。
4. 幻觉校验 Agent 快速抽查今日新卡和长题；未通过者替换为已校验内容。
5. 系统向用户提问：先主动回忆，后填空，再推导或综合题；答案默认不提前暴露。
6. 用户逐题作答后，批改讲解 Agent 给出评级、讲解和必要追问。
7. 复习调度 Agent 根据评级更新间隔、到期日和当日日志。
8. 出题变式 Agent 针对 `again` 和 `hard` 题补 1-3 个同类小变式。
9. 总规划 Agent 汇总当日完成度、薄弱点和明日建议。
10. 资料归档 Agent 归档日志、更新产物状态，并列出待解析或待校验缺口。

## 大型抽卡流程

1. 总规划 Agent 按章节拆批次，限制每批卡片数量，避免一次生成过大。
2. 教材解析 Agent 先输出目录、公式、例题索引和来源问题。
3. 数学抽卡 Agent 与信号抽卡 Agent 按学科并行生成 `draft` 卡片。
4. 幻觉校验 Agent 对每批卡片核查来源、条件、答案和版权风险。
5. 原抽卡 Agent 修复 `needs_fix` 项；无法核实的保留为 `unverified`，不入正式复习。
6. 出题变式 Agent 只基于 `checked` 卡片生成练习题。
7. 资料归档 Agent 记录批次、来源、卡片状态和缺口。

## 最小可用运行

当没有 subagent 或任务很小时，由当前 Codex 依次扮演角色：

1. 用总规划 Agent 的视角限定范围。
2. 用教材解析 Agent 的视角确认来源。
3. 用抽卡或出题 Agent 生成小批量草稿。
4. 用幻觉校验 Agent 自检并标记不确定项。
5. 用复习调度和批改讲解 Agent 完成当日复习闭环。

即使单 agent 执行，也要保留角色化交接信息，方便后续并行 agent 接手。
