# PDF 解析与 OCR 质检

在从教材、讲义、真题 PDF 抽卡前，先运行 PDF 解析与 OCR 质检流程。目标不是“尽快抽出文字”，而是判断哪些内容可靠、哪些内容必须回查。

## 质检顺序

1. 建立文件清单：
   - 路径、文件名、大小、页数、资料类别。
   - 标记来源文件夹：教材、基础、强化、真题、解析。

2. 判断 PDF 类型：
   - `native_text`：可复制文本，章节标题和普通文字基本正常。
   - `scanned_image`：整页图片，需要 OCR。
   - `mixed`：部分页面有文本层，公式或例题为图片。
   - `corrupt_or_locked`：无法读取、加密、缺页或工具报错。

3. 抽样检查：
   - 首页、目录页、每章开头、公式密集页、例题页、答案解析页。
   - 数学资料至少检查：上下标、积分限、求和限、矩阵、分段函数、极限符号。
   - 信号资料至少检查：傅里叶/拉普拉斯/Z 变换符号、卷积、冲激函数、ROC、系统函数。

4. 建立页码映射：
   - `pdf_page` 是 PDF 文件页序。
   - `print_page` 是书页印刷页码。
   - 若目录页、封面、版权页导致错位，记录偏移。
   - 无法确认印刷页码时写 `TBD`，不要猜。

5. 输出质检结论：
   - `high`：文本层可靠，抽样公式和例题编号无明显问题。
   - `medium`：正文可用，但公式、页码、例题号需要抽样复核。
   - `low`：扫描模糊、OCR 错误多、公式不可直接信任。
   - `blocked`：无法读取或缺关键页面。

## 低置信度规则

以下内容即使 OCR 出了文本，也必须二次核验：

- 公式中的上下标、积分上下限、求和范围、矩阵元素和转置/逆符号。
- 例题编号、定理编号、页码、章节编号。
- 分段函数、绝对值、向量/矩阵粗体、复数单位、频域变量。
- 信号与系统中的 `u(t)`、`δ(t)`、`X(jω)`、`H(s)`、`X(z)`、ROC 条件。
- 扫描版 PDF 的表格、图像题、跨页题目、答案解析中的换行公式。

## Agent 输出模板

```markdown
## PDF 质检报告
- 文件：
- 类型：native_text/scanned_image/mixed/corrupt_or_locked
- 页数：
- 质量等级：high/medium/low/blocked
- 页码映射：
- 可直接抽取范围：
- 必须回查范围：
- 建议策略：
```

## 与其他 Agent 的关系

- PDF 解析与 OCR 质检 Agent 先于教材解析 Agent。
- 教材解析 Agent 只能把 `high` 内容直接作为可抽取来源。
- `medium` 内容可以生成草稿卡，但必须附 `source_status: needs_review`。
- `low` 内容只能列入待人工确认清单，除非用户明确接受低置信度草稿。
- 幻觉校验 Agent 优先审查 `medium` 和 `low` 区域。
